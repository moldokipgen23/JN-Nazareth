Ehlom Blog CMS Documentation

Thank you for purchasing Ehlom Blog CMS. This guide will walk you through installation, configuration, and day-to-day use of your new website.

Who is this for? Schools (showcase staff, alumni, events, news), churches (services, ministries, sermons, congregation), NGOs (mission, programmes, impact, donate CTA), community groups and villages (members, activities, photo galleries), and anyone who wants a polished blog-style website with no coding required.

Ehlom Blog CMS is a complete content management system built for schools, churches, NGOs, community groups, villages, and any organisation that needs a blog-style website. Whether you run a school, a faith group, a non-profit, a club, or a community association, this CMS gives you everything you need. Everything on the website — colours, logo, text, sections, photos — is editable from the admin panel without touching code.

1. Server Requirements

Ehlom Blog CMS runs on any standard shared or VPS hosting that supports modern PHP. Most cPanel hosts (Hostinger, Namecheap, SiteGround, Bluehost, A2 Hosting, etc.) meet these requirements out of the box.

Minimum Requirements

ComponentRequired
PHP8.2 or higher (8.3 recommended)
DatabaseMySQL 5.7+ / MariaDB 10.3+ or SQLite 3.8+
Web ServerApache (with mod_rewrite) or Nginx
Disk Space200 MB minimum (500 MB recommended)
Memory Limit128 MB minimum (256 MB recommended)

Required PHP Extensions

  • PDO & pdo_mysql or pdo_sqlite
  • OpenSSL
  • Mbstring
  • Tokenizer
  • XML
  • Ctype
  • JSON
  • BCMath
  • Fileinfo
  • GD or Imagick (for image processing)
Tip: If you are unsure whether your host supports these, the web installer at /install.php will check automatically and tell you what is missing.

2. Installation

Installation takes about 5 minutes. No terminal commands required.

1Upload the Files

Unzip the purchased package on your computer. Inside you will find a source/ folder — this is your website.

Using cPanel File Manager or FTP (FileZilla):

  • Upload all files from source/ into your hosting account.
  • If installing on the main domain, upload to public_html/.
  • If installing in a subdirectory, upload to public_html/yoursubfolder/.
Important: Set your domain's document root to the /public folder of the upload. In cPanel this is done under Domains → Manage → Document Root. If you cannot change the document root, see the subfolder workaround.

2Create a Database (MySQL only)

If you want to use MySQL (recommended for production), create a database first:

  1. Log into cPanel.
  2. Open MySQL Databases.
  3. Create a new database (e.g. yourname_ehlom).
  4. Create a database user with a strong password.
  5. Add the user to the database with ALL PRIVILEGES.
  6. Note the database name, username, and password — you will need them in step 3.

If you prefer SQLite, you can skip this step entirely. SQLite needs no setup.

3Run the Web Installer

Open your browser and visit:

https://yourdomain.com/install.php

The installer will guide you through four screens:

  1. Requirements Check — green ticks mean you are good to go.
  2. Database & Site Info — choose SQLite or MySQL, enter your site name and details.
  3. Admin Account — create your first admin user (email + password).
  4. Finalising — the installer runs migrations, seeds default content, creates the storage symlink, and caches the config.

When it finishes, you will see a success page with links to your homepage and admin panel.

4Delete the Installer (Security)

After a successful install, delete install.php from your server (both the root one and public/install.php). This prevents anyone from re-running it.

If You Cannot Change the Document Root

Some shared hosts do not let you point the domain to /public. In that case:

  1. Move everything inside /public up one level into public_html/.
  2. Open index.php and update the two require paths from __DIR__.'/../' to __DIR__.'/'.
If you are not comfortable doing this, contact your hosting provider's support — they can change the document root for you in 2 minutes.

4. Customization

Everything visual is editable. Go to Admin → Site Customizer — you will see tabs for each area.

Branding Tab

  • Site name — appears in the header, footer, browser tab, and emails.
  • Tagline — short description shown in the header and on social previews.
  • Logo — upload PNG/JPG/SVG. Recommended 200×60 px transparent PNG.
  • Favicon — small icon for browser tabs. Use a 32×32 or 64×64 PNG/ICO.

Colours Tab

  • Primary colour — main theme colour (buttons, links, accents).
  • Secondary colour — used for highlights and CTAs.
  • Pick any hex value or use the colour picker. Changes apply immediately after saving.

Contact Tab

Address, phone, email, and Google Maps embed code — all displayed on the Contact page and in the footer.

Social Tab

Facebook, Twitter/X, Instagram, YouTube, WhatsApp — leave blank to hide the icon.

Footer Tab

  • Footer description text.
  • Column headings ("Quick Links", "Contact Us") — fully customisable.
  • Copyright line.
  • Optional credit text ("Designed by Ehlom Digital") — toggle on/off.

SEO Tab

Meta title, meta description, and keywords for search engines. Add Google Analytics tracking ID here.

6. Managing Content

Writing a Blog Post

  1. Go to Admin → Blog Posts → Add New.
  2. Enter a title, choose a category, and write your content using the rich-text editor.
  3. Upload a featured image (recommended 1200×630 px).
  4. Set status to Published and click Save.

Adding an Event

  1. Go to Admin → Events → Add New.
  2. Set the title, date/time, location, and description.
  3. Upload a poster image (optional).
  4. Save. The event appears in the Events section and Events page automatically.

Uploading Gallery Images

  1. Go to Admin → Gallery.
  2. Create a folder (e.g. "Annual Gathering 2025", "Volunteer Day").
  3. Click the folder and upload images — you can drag-drop multiple files.

Managing Hero Slides

  1. Go to Admin → Banner Slides.
  2. Add up to 5 slides with an image, headline, subtext, and optional button link.
  3. Reorder by dragging.

7. Users & Roles

Ehlom Blog CMS has three built-in roles:

RoleWhat They Can Do
AdminFull access — settings, users, all content.
StaffCreate & edit content (blog, events, gallery). Cannot change settings or manage users.
ViewerRead-only access to the admin dashboard.

To add a new user: Admin → Users → Add New. Choose their role from the dropdown.

8. Updating to a New Version

When a new version of Ehlom Blog CMS is released:

  1. Back up your files and database first. Use cPanel's backup tool or export the database from phpMyAdmin.
  2. Download the new version from your CodeCanyon downloads page.
  3. Upload the new files, overwriting the old ones. Do not overwrite the .env file or the storage/ folder.
  4. Visit yourdomain.com/admin — if the new version requires database changes, you will be prompted to apply them.
Always back up before updating. If anything goes wrong, you can restore in minutes.

9. Troubleshooting

500 Internal Server Error after install

Usually a file permissions issue. Set storage/ and bootstrap/cache/ to 775 (or 755 on stricter hosts). In cPanel File Manager, right-click each folder → Change Permissions.

Uploaded images don't show

The storage symlink may not have been created. Visit yourdomain.com/install.php?step=fix-storage to recreate it, or contact support.

"Page not found" on every link except homepage

Apache's mod_rewrite is not enabled, or .htaccess is being ignored. Contact your host to enable mod_rewrite and AllowOverride All.

Admin panel blank or broken styling

Clear the cache: Admin → Settings → Clear Cache. If that fails, delete the contents of bootstrap/cache/ (but not the folder itself).

Forgot admin password

Use the Forgot password? link on the login page — it will email a reset link. If email is not configured, contact support for a manual reset script.
